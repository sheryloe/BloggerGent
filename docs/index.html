Quick Answer

Run Blogger content operations without blind auto-publishing.

Bloggent is a multi-blog Blogger workspace that combines AI-assisted topic discovery, article generation, image generation, inline post review, manual publishing, and live metadata verification. It is built for operators who want speed, but still want the last call before anything public goes out.

At a Glance

What the product does right now

  • One dashboard for content actions, queue visibility, full preview, and metadata state.
  • Blog-specific workflows and prompts for different content identities.
  • Inline review in /articles before manual Blogger release.
  • Verification for description, og:description, and twitter:description.
  • Docker-first local stack with Next.js, FastAPI, Celery, PostgreSQL, and Redis.
Why it exists

Most Blogger tools stop at generation. Operators need the review and verification layers too.

Bloggent is opinionated about the risky parts of publishing. It treats generation, publishing, and public metadata verification as separate moments because those are the moments where bad automation usually breaks trust.

Content ops

Topic to post flow

Discover topics, generate the article package, create imagery, assemble HTML, then review the real post shape before release.

Safety

Manual by design

Publishing is intentionally separate from generation so weak drafts do not become public just because a model responded.

Verification

Live metadata checks

Bloggent verifies the public page instead of assuming Blogger metadata fields turned into real head tags.

Workflow

The product loop is simple for operators, but the pipeline underneath stays flexible per blog.

Different blogs can keep different prompt systems and category rules while the operator still sees one clear review path.

Operator path

What the user sees

  • Choose a blog and enter a topic or run discovery.
  • Generate the article package and review it in /articles.
  • Preview the fully assembled post inline, not inside a cramped inner frame.
  • Publish manually and verify the live metadata afterwards.
System path

What the platform manages

  • Prompt rendering per blog workflow.
  • AI article generation and optional topic discovery.
  • Image generation and HTML assembly.
  • Publish-state tracking, queueing, and verification support.
GEO + SEO

The writing system is now answer-first, metadata-aware, and easier for both search engines and answer engines to quote.

Bloggent's current article prompts are built around direct openings, quote-ready section starts, metadata alignment, and structure that makes the final article easier to summarize and easier to trust.

Prompt structure

Answer early

The first part of the article should answer the real query fast, name the entity early, and explain practical value before long scene-setting.

Metadata

Align the promise

meta_description, excerpt, and the opening body copy should promise the same thing instead of pulling in different directions.

Blogger reality

Verify the live page

The dashboard meta state is not a body-quality score. It is a live verification state for the public description tags the crawler may actually see.
