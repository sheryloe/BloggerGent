빠른 답변

글 생성은 빠르게, 공개는 신중하게. Bloggent는 그 흐름을 위한 Blogger 운영 워크스페이스입니다.

Bloggent는 여러 Blogger 블로그를 하나의 작업 공간에서 관리하면서, AI 주제 발굴, 글 생성, 이미지 생성, 최종 포스트 프리뷰, 수동 발행, 공개 메타 검증까지 이어지는 실제 운영 흐름을 정리해 주는 도구입니다. 자동화는 하되, 발행 전 마지막 판단은 운영자가 직접 하도록 설계했습니다.

지금 기준으로 Bloggent가 해주는 것

  • 대시보드 한 화면에서 글 작성, 작업 큐, 실제 포스트 프리뷰, 메타 상태를 함께 확인할 수 있습니다.
  • 블로그마다 다른 워크플로와 프롬프트를 유지해 채널 정체성을 분리합니다.
  • /articles에서 최종 글 모양을 검토한 뒤 수동으로 Blogger에 발행합니다.
  • description, og:description, twitter:description 메타를 실제 공개 페이지 기준으로 검증합니다.
  • Next.js, FastAPI, Celery, PostgreSQL, Redis 기반의 Docker 중심 로컬 스택으로 바로 띄울 수 있습니다.
왜 만들었나

대부분의 Blogger 도구는 생성까지만 다룹니다. 실제 운영에는 검토와 검증 단계가 더 중요합니다.

Bloggent는 생성, 발행, 메타 검증을 서로 다른 순간으로 나눕니다. 위험한 초안이 바로 공개되거나, Blogger 메타가 제대로 반영됐는지 모른 채 넘어가는 상황을 줄이기 위해서입니다.

운영 흐름

주제부터 포스트까지

주제 발굴, 글 생성, 이미지 생성, HTML 조립까지 이어진 뒤, 운영자가 실제 공개 형태를 보고 마지막 검토를 할 수 있습니다.

안전장치

생성과 발행의 분리

모델이 글을 만들었다고 바로 공개하지 않습니다. 생성과 발행을 분리해서 공개 전 검토 시간을 확보합니다.

검증

실제 공개 페이지 기준 확인

Blogger API가 메타를 받았다는 이유만으로 끝내지 않고, 공개 페이지에서 실제 메타 태그가 어떻게 보이는지 확인합니다.

워크플로

운영자에게는 단순하게, 내부 파이프라인은 블로그별로 유연하게 유지하는 구조입니다.

여행 블로그와 미스터리 블로그는 같은 말투와 구조를 쓰면 안 됩니다. Bloggent는 블로그마다 다른 프롬프트와 카테고리 규칙을 유지하면서도, 운영자 화면에서는 한눈에 보이는 검토 흐름을 제공합니다.

운영자 관점

운영자가 보는 흐름

  • 블로그를 고르고 주제를 직접 넣거나 AI 주제 발굴을 실행합니다.
  • 생성된 글 패키지를 /articles에서 검토합니다.
  • 작은 iframe이 아니라 실제 조립된 포스트를 페이지 안에서 바로 확인합니다.
  • 수동 발행 후 공개 페이지 메타 상태를 다시 검증합니다.
시스템 관점

시스템이 관리하는 흐름

  • 블로그별 워크플로에 맞춘 프롬프트 렌더링
  • AI 글 생성과 필요시 주제 발굴
  • 이미지 생성과 HTML 조립
  • 발행 상태 추적, 작업 큐, 메타 검증 보조
GEO + SEO

현재 글 생성 프롬프트는 답을 먼저 주고, 메타 약속을 맞추며, 검색과 답변형 엔진 모두가 읽기 쉽게 구성됩니다.

핵심 질문에 빠르게 답하고, 각 섹션이 실제 하위 질문을 해결하며, 메타 설명과 본문 첫 부분이 같은 약속을 하도록 맞추는 것이 현재 Bloggent 글 구조의 기본 원칙입니다.
