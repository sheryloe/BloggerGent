Service concept

블로그 여러 개를 사람 손이 닿는 서비스처럼 운영하기 위한 워크스페이스

Bloggent는 “글을 하나 만들어주는 툴”보다, “블로그 여러 개를 사고 없이 운영하는 서비스”에 가깝습니다. 생성, 이미지, HTML 조립, 수동 발행, SEO 검증을 분리해서 운영자가 마지막 판단을 하도록 설계했습니다.

저장소 보기 이슈 확인

Why it exists

블로그마다 프롬프트와 말투가 다르고, 발행은 마지막에 사람이 확인해야 합니다.

다중 블로그 운영

블로그별 워크플로와 프롬프트를 분리합니다.

발행 보호

공개 포스트 덮어쓰기를 막고, 수동 발행을 유지합니다.

콘텐츠 생성

토픽 탐색, 초안 작성, 이미지 생성, HTML 조립이 같은 흐름으로 이어져야 합니다.

안전한 발행

생성과 발행을 분리하고, 공개 포스트 보호와 중복 주제 차단이 필요합니다.

실제 SEO 확인

API 메타데이터가 아니라 공개 페이지에서 진짜 노출 상태를 봐야 합니다.

Pipeline

Bloggent가 풀고 있는 운영 루프

01 Import

Blogger 블로그와 데이터를 가져옵니다.

02 Configure

블로그별 프롬프트와 워크플로를 분리합니다.

03 Generate

토픽, 글, 이미지, HTML을 생성합니다.

04 Review

생성 결과를 검토하고 발행 대기 상태로 둡니다.

05 Verify

공개 페이지 기준 SEO 상태를 재검증합니다.

Product

예약 발행 캘린더

운영 서비스로 가려면 생성 결과를 시간표 기준으로 관리할 수 있어야 합니다.

Workflow

승인 큐와 QA

발행 버튼 전 단계에서 제목, 메타, 이미지, 중복성 체크가 필요합니다.

Growth

다중 운영자 모델

사용량, 역할, 블로그별 접근 범위를 나누면 서비스화 가능성이 크게 올라갑니다.
