GEO + SEO Agent Platform

AI 글 생성에서 끝나지 않는 Blogger 운영 플랫폼

Bloggent는 여러 Blogger 블로그를 하나의 워크스페이스에서 운영하면서, 주제 발굴, 글 패키지 생성, 이미지 흐름, 수동 발행, 공개 메타 검증까지 실제 운영 루프를 한 제품 안에서 연결합니다. 자동화는 빠르게, 공개 결정은 신중하게 가져가고 싶은 운영자를 위한 구조입니다.

Why Bloggent

초안 생성기 하나로는 부족한 운영 문제를 제품 레이어에서 해결합니다

대부분의 블로그 자동화 도구는 글 생성까지만 다룹니다. 하지만 실제 운영에서는 검토, 발행 안전성, 블로그별 정체성 분리, 공개 메타 반영 여부 확인이 더 중요합니다. Bloggent는 그 빈칸을 채우기 위해 설계되었습니다.

문제

모든 블로그에 같은 프롬프트를 쓸 수 없습니다

여행 블로그와 미스터리 블로그는 독자의 기대와 문장 구조가 다릅니다. Bloggent는 블로그별 워크플로와 프롬프트 체인을 따로 유지합니다.

문제

생성과 발행이 붙어 있으면 운영 리스크가 커집니다

AI가 글을 만들었다고 바로 공개하면 품질 편차와 메타 누락이 그대로 노출됩니다. Bloggent는 공개 결정을 운영자에게 남깁니다.

문제

Blogger 메타는 입력보다 검증이 더 중요합니다

customMetaData 입력만으로 공개 <head> 반영이 보장되지는 않습니다. 그래서 Bloggent는 후검증 루프를 둡니다.

Platform

주제 발굴, 글 생성, 검토, 발행 검증을 하나의 서비스 흐름으로 묶었습니다

Bloggent의 핵심은 에이전트를 많이 붙이는 것이 아니라, 운영자가 필요한 판단만 빠르게 할 수 있도록 단계와 화면을 분리하는 데 있습니다.

Discovery Layer

시간축이 있는 주제 발굴

블로그 카테고리와 현재 날짜를 바탕으로, 실제 수요가 생길 가능성이 높은 키워드를 우선 발굴합니다. 시즌성과 일정성이 중요한 채널에 특히 효과적입니다.

Writing Layer

GEO + SEO 글 패키지

제목, 메타 설명, excerpt, 본문, FAQ, 이미지 문맥이 따로 놀지 않도록 한 번에 생성합니다. 검색엔진과 답변형 엔진 모두를 고려한 구조입니다.

Release Layer

수동 발행 보호와 후검증

발행은 버튼 하나지만, 그 결정은 검토 화면에서 이뤄져야 합니다. Bloggent는 공개 전 프리뷰와 공개 후 메타 검증을 나란히 제공합니다.

Operator Flow

운영자는 이렇게 움직입니다

  1. 블로그를 선택하고 직접 주제를 넣거나 AI 주제 발굴을 실행합니다.
  2. 백그라운드 작업 큐에서 글 패키지와 관련 산출물을 생성합니다.
  3. /articles에서 제목, 메타 설명, excerpt, HTML, 최종 프리뷰를 검토합니다.
  4. 최종 판단이 끝나면 수동으로 발행합니다.
  5. 발행 후 공개 메타 상태를 다시 검증해 실제 페이지 반영 결과를 확인합니다.
Product Surfaces

화면은 운영 판단이 빠르도록 설계됩니다

  • / 대시보드: 글 시작 액션, 작업 큐, 대표 프리뷰, 메타 상태 요약
  • /articles: 최종 검토, 수동 발행, 공개 메타 확인을 위한 핵심 화면
  • /jobs: 실패, 재시도, 백그라운드 상태 가시화
  • /settings: Google OAuth, Blogger 가져오기, Search Console/GA4 매핑, 워크플로 설정
Documentation Hub

제품 소개와 실무 가이드를 문서 허브와 위키로 함께 관리합니다

GitHub Pages 랜딩은 제품 소개 허브 역할을, 위키는 운영 문서 허브 역할을 맡습니다. 처음 보는 사람은 아래 문서부터 읽으면 바로 로컬 실행 루프까지 이어질 수 있습니다.
