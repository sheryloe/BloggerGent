# Step 1. 구글 블로그 반자동화 프로젝트를 시작한 이유와 키워드 전략

## 추천 제목
- 구글 블로그 자동화, 왜 직접 만들었나? 키워드 전략부터 정리한 실전 기록
- Blogger SEO 자동화 프로젝트 시작기: 문제 정의와 키워드 설계
- 구글 블로그를 반자동화한 이유: 콘텐츠 운영 문제와 SEO 키워드 전략

## 검색 설명
구글 블로그 반자동화 프로젝트를 시작한 이유와 키워드 전략을 정리합니다. Blogger 운영 자동화의 핵심 문제와 SEO 방향을 함께 다룹니다.

## 추천 태그
`구글블로그자동화`, `BloggerSEO`, `블로그자동화`, `콘텐츠운영`, `키워드전략`

## 글 구조
- 왜 이 프로젝트를 시작했는가
- 기존 Blogger 운영의 반복 업무
- 자동화로 줄이고 싶은 작업
- 어떤 키워드 카테고리로 운영할 것인가
- 여행/축제 블로그와 미스터리 블로그를 왜 분리했는가

## 본문 샘플

반자동 블로그 운영 시스템을 직접 만든 이유는 단순했습니다. 글 한 편을 올릴 때마다 주제 정리, 제목 작성, 이미지 연결, HTML 구성, 게시, 성과 확인까지 반복해야 했기 때문입니다.

특히 Google Blogger는 가볍고 빠르지만, 운영을 오래 하다 보면 사람 손이 들어가는 작업이 많습니다. 그래서 이번 프로젝트의 목표는 완전 자동이 아니라 **반자동 운영 흐름을 한 화면으로 묶는 것**이었습니다.

## HTML 예시

```html
<h2>왜 Blogger 자동화가 필요했나</h2>
<p>Blogger는 여전히 강력한 퍼블리싱 도구이지만, 반복 운영 단계가 많아질수록 사람의 시간이 많이 들어갑니다.</p>
<p>주제 정리, 글 작성, 대표 이미지, HTML 조립, 게시, 성과 확인이 따로 놀면 운영 효율이 급격히 떨어집니다.</p>

<h2>이번 프로젝트의 핵심 목표</h2>
<ul>
  <li><strong>주제 입력</strong>만으로 초안이 생성될 것</li>
  <li><strong>대표 이미지</strong>가 자동으로 공개 URL을 가질 것</li>
  <li><strong>Blogger 게시</strong>와 <strong>성과 확인</strong>이 한 화면에서 가능할 것</li>
</ul>
```

## 마무리 문장 예시

이 프로젝트의 시작점은 화려한 AI 기능이 아니라, 사람이 매번 반복하던 블로그 운영 단계를 줄이는 것이었습니다. 다음 글에서는 이 흐름을 실제 서비스처럼 나누기 위해 어떤 구조로 아키텍처를 설계했는지 정리해보겠습니다.
