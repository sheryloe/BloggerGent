# Step 1. Blogger 자동화 서비스를 왜 다시 설계했는가

## 추천 제목

1. Blogger 자동화 서비스를 다시 만든 이유, 글 생성보다 운영이 더 어려웠던 문제
2. AI 블로그 자동화의 함정: 글 생성만으로는 서비스가 되지 않는 이유
3. Blogger 반자동 운영 시스템을 설계하며 정리한 핵심 문제 5가지

## 검색 설명

AI로 글을 쓰는 것과 실제 블로그 서비스를 운영하는 것은 다르다. Blogger 자동화 시스템을 다시 설계하며 마주친 문제와 해결 방향을 정리했다.

## 본문

블로그 자동화라는 말을 들으면 많은 사람이 가장 먼저 떠올리는 것은 “주제 하나 넣으면 글이 자동으로 써지는 도구”일 것이다. 실제로 나도 처음에는 그 단계만 잘 만들면 충분할 것이라고 생각했다. 주제를 만들고, 본문을 생성하고, 이미지를 붙이고, API로 발행하면 끝이라고 봤다. 그런데 막상 여러 개의 블로그를 동시에 운영하려고 보니 문제는 전혀 다른 곳에서 터졌다.

첫 번째 문제는 블로그마다 성격이 완전히 다르다는 점이다. 한국 여행과 행사 정보를 영어로 소개하는 블로그와, 미스터리 실화나 전설을 영어로 풀어내는 블로그는 문체도 다르고 구조도 다르다. 같은 프롬프트를 공유하면 결과가 금방 밋밋해지고, 특정 블로그의 색이 사라진다. 그래서 공용 프롬프트 하나로 모든 채널을 돌리는 방식은 생각보다 빨리 한계에 부딪힌다.

두 번째 문제는 생성과 게시를 같은 단계로 묶으면 사고가 난다는 점이다. 초안이 완성되자마자 바로 발행되는 구조는 언뜻 편해 보이지만, 실제 운영에서는 가장 위험하다. 특히 Blogger는 한 번 공개된 글을 다시 잘못 업데이트하면 기존 검색 유입 글이 덮어써질 수 있다. 그래서 실제 서비스라면 생성과 게시를 분리하고, 마지막 게시만큼은 사람이 한 번 더 확인하는 구조가 안전하다.

세 번째 문제는 SEO 메타데이터다. 많은 사람이 Blogger API의 `customMetaData` 필드를 보면 “여기에 메타 설명을 넣으면 되겠구나”라고 생각한다. 나도 처음에는 그렇게 접근했다. 하지만 실제 공개 페이지의 `<head>`를 보면 원하는 `description`, `og:description`이 그대로 반영되지 않는 경우가 많다. 즉 DB에 값이 있고 API로 전송했다고 해서 공개 페이지 SEO가 완성되는 것이 아니다. 이 지점에서 단순 글 생성 도구와 실제 운영 도구의 차이가 명확하게 드러난다.

네 번째 문제는 운영 정보의 분산이다. 주제는 어디서 생성했고, 어떤 프롬프트로 썼고, 어떤 이미지가 붙었고, 지금 공개 상태는 어떤지, Google Search Console과 GA4 데이터는 연결됐는지까지 한 화면에서 보고 싶었다. 그런데 대부분의 자동화 도구는 글 생성만 하고 끝난다. 내가 원했던 것은 “콘텐츠 생성기”가 아니라 “블로그 운영 콘솔”에 더 가까웠다.

그래서 BloggerGent는 방향을 다시 잡았다. 목표를 글쓰기 자동화에서 서비스형 블로그 반자동 운영으로 바꿨다. 핵심은 아래 네 가지였다.

- 블로그마다 다른 워크플로와 프롬프트를 갖게 할 것
- 생성과 게시를 분리할 것
- 공개 글 덮어쓰기를 막을 것
- 실제 공개 페이지 기준 SEO 검증까지 포함할 것

이렇게 정리하고 나니 프로젝트의 성격이 훨씬 선명해졌다. 더 이상 “AI가 글 써주는 툴”이 아니라, 여러 개의 Blogger 채널을 운영할 때 필요한 파이프라인과 검증 단계를 한 번에 묶는 서비스가 된 것이다. 이 시리즈에서는 그 과정에서 정리한 구조, 코드, 설정, 운영 포인트를 단계별로 풀어보려고 한다.

```html
<h2>왜 다시 설계했는가</h2>
<p>블로그 자동화의 핵심은 글 생성이 아니라 운영 안정성이다. 특히 여러 블로그를 함께 돌릴 때는 프롬프트 분리, 게시 보호, SEO 검증이 반드시 필요하다.</p>

<h2>핵심 문제</h2>
<ul>
  <li>블로그마다 문체와 구조가 다르다</li>
  <li>생성과 게시를 묶으면 공개 글이 덮어써질 수 있다</li>
  <li>Blogger API 메타데이터는 공개 SEO 태그와 다를 수 있다</li>
  <li>운영 정보를 한 화면에서 보기 어렵다</li>
</ul>
```
