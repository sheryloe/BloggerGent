# Step 6. 대시보드와 글 검토 UX를 왜 다시 다듬었는가

## 추천 제목

1. 블로그 자동화 도구에서 대시보드 UX가 중요한 이유
2. 생성된 글 목록, 작업 목록, 설정 화면을 왜 다시 정리했는가
3. AI 블로그 운영 도구는 예쁜 화면보다 사고를 줄이는 UX가 중요하다

## 검색 설명

대시보드와 글 목록, 설정 화면을 단순히 보기 좋게가 아니라 운영 사고를 줄이는 방향으로 다시 설계한 이유와 기준을 설명한다.

## 추천 태그

`#대시보드UX #운영도구설계 #NextjsUI #블로그자동화UI #글검토흐름`

## 본문

운영 도구의 UI는 일반 소비자 앱과 기준이 다르다. 처음에는 카드가 화려하고 정보가 많으면 좋아 보인다. 하지만 실제로 매일 열어보는 화면이 되면, 중요한 것은 화려함이 아니라 “지금 어디를 봐야 하는지 바로 알 수 있느냐”다. BloggerGent를 만들면서 이 사실을 여러 번 체감했다.

대표적인 예가 생성된 글 목록이다. 예전에는 글 상세와 카드 정보가 한 화면에서 복잡하게 섞여 있어서 읽기가 불편했고, 실제 Blogger 공개 글과 대시보드 미리보기의 색감까지 충돌하는 문제가 있었다. 그래서 글 목록은 제목 리스트를 먼저 보여주고, 선택한 글의 상세를 아래에서 보는 방식으로 단순화했다. 운영자는 한 번에 하나의 글만 집중해서 검토하면 충분하기 때문이다.

작업 목록도 마찬가지였다. 작업 화면은 로그를 잔뜩 보여주는 것보다, 어떤 작업이 성공했고 어떤 작업이 실패했는지, 지금 초안 상태인지 게시 상태인지가 더 중요하다. 그래서 숫자와 상태를 명확히 나누고, 최신 작업을 보기 쉬운 흐름으로 다시 정리했다.

설정 화면은 더 복잡하다. API 키, Google OAuth, Blogger import, 블로그별 워크플로, 프롬프트, SEO 메타 패치 같은 내용이 한곳에 모여 있기 때문이다. 여기서 중요한 것은 “기능을 많이 넣는 것”이 아니라 “순서를 이해시키는 것”이다. 그래서 설정 화면도 전역 연결, Blogger/Google 연결, 블로그별 워크플로, 프리셋 라이브러리처럼 역할을 나눠 보여주는 방식이 훨씬 낫다고 판단했다.

특히 글 공개 버튼은 설정 화면이 아니라 글 목록에 두는 것이 맞다고 봤다. 게시는 설정이 아니라 결과물 검토 이후의 최종 액션이기 때문이다. 이 작은 차이가 공개 글 덮어쓰기 같은 사고를 줄이는 데 꽤 크게 작용했다.

결국 운영 도구 UX의 핵심은 “빠른 이해”와 “안전한 행동 유도”다. 예쁜 인터페이스보다도, 잘못 누르기 어렵고 상태를 오해하지 않게 만드는 구성이 훨씬 중요하다. BloggerGent의 대시보드도 그 기준으로 조금씩 정리되고 있다.

## HTML 예시

```html
<h2>글 목록 UX 원칙</h2>
<ul>
  <li>제목 리스트를 먼저 보여준다</li>
  <li>선택한 글의 상세를 아래에서 검토한다</li>
  <li>최종 공개 버튼은 글 상세에 둔다</li>
</ul>

<h2>왜 이렇게 바꿨는가</h2>
<p>운영 도구는 정보가 많아 보이는 것보다, 잘못 클릭하지 않게 설계하는 것이 더 중요하다.</p>
```
