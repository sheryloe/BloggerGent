# Step 6. 설정 화면과 운영 대시보드 UX 정리

## 추천 제목
- 블로그 자동화 툴은 UX가 핵심이다: 설정 화면과 운영 대시보드 개선기
- Blogger 자동화 대시보드 UX 설계: 설정, 미리보기, 모니터링 정리
- AI 블로그 운영 툴을 서비스답게 만들기 위한 화면 구성 정리

## 검색 설명
블로그 import, 전역 설정, 블로그별 워크플로, HTML 미리보기, Google 데이터 모니터링까지 서비스형 UX로 정리한 과정을 다룹니다.

## 추천 태그
`대시보드UX`, `관리자화면`, `서비스설계`, `NextjsUI`, `운영툴`

## 글 구조
- 왜 설정 화면을 다시 나눴는가
- 왜 블로그별 탭 구조가 필요했는가
- 왜 HTML 미리보기를 좌우 분리형으로 바꿨는가
- 왜 Google 데이터 화면을 따로 만들었는가

## 본문 샘플

초기 버전은 기능은 있었지만, 실제 운영자가 쓰기에는 정보가 너무 한 화면에 몰려 있었습니다. 특히 블로그가 여러 개일 때 세로로만 쌓이는 구조는 금방 관리가 어려워졌습니다.

그래서 블로그 목록은 왼쪽, 상세 설정은 오른쪽으로 나누고, 상세는 다시 `연결 / 기본 정보 / 파이프라인 / 모니터링` 탭으로 정리했습니다. 이 구조 덕분에 한 계정에 블로그가 여러 개 있어도 훨씬 관리가 쉬워졌습니다.

## HTML 예시

```html
<h2>설정 화면을 어떻게 나눴나</h2>
<ul>
  <li><strong>전역 설정</strong>: AI 키, Google OAuth, 이미지 공개 방식</li>
  <li><strong>블로그 목록</strong>: 가져온 Blogger 블로그 선택</li>
  <li><strong>상세 탭</strong>: 연결 / 기본 정보 / 파이프라인 / 모니터링</li>
</ul>

<h2>왜 HTML 미리보기를 바꿨나</h2>
<p>기존에는 여러 HTML 카드가 겹쳐 보여서 확인이 어려웠습니다.</p>
<p>지금은 왼쪽에서 글을 고르고 오른쪽에서 한 편씩 크게 미리보는 구조로 정리했습니다.</p>
```

## 마무리 문장 예시

운영 툴은 기능만 있는 것으로 끝나지 않습니다. 실제로 다른 사람도 쓸 수 있으려면 설정 흐름과 안내 문서까지 정리되어 있어야 합니다. 마지막 글에서는 운영 마무리와 SEO 관점에서 정리한 포인트를 다뤄보겠습니다.
