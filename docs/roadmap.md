# 로드맵

이 문서는 Bloggent의 현재 제품 방향과 TODO 우선순위를 정리합니다.

## 빠른 답변

지금 Bloggent의 로드맵은 “기능을 더 많이 자동화하기”보다 “운영자가 더 빠르게 보고, 더 안전하게 발행하게 만들기”에 초점을 둡니다.

## 이미 들어간 것

- 멀티 블로그 가져오기와 블로그별 워크플로 분리
- 수동 발행 보호 흐름
- 생성 글 검토 화면과 최종 포스트 프리뷰
- GEO + SEO 글 생성 프롬프트 구조
- 공개 메타 검증 흐름

## Now

- Blogger 테마 메타 패치 안내 흐름 개선
- 대시보드 액션 이후 체감 속도 개선
- 남은 화면들의 UI와 문구 톤을 운영형 제품답게 통일
- 글 단위와 블로그 단위 메타 상태를 더 쉽게 이해하게 만들기

## Next

- 제목, 메타, 이미지, 중복 기준의 발행 전 QA 체크 추가
- Search Console과 GA4 가시성 확장
- 운영자 승인 유지형 예약 발행 추가
- 실패 작업과 재시도 이력 가시성 개선

## Later

- 멀티 유저 권한 관리
- 웹 앱의 프로덕션 실행 프로파일 보강
- 공개 에셋 공급자 확장
- 카테고리별 프롬프트 팩 확장

## 제품 원칙

Bloggent는 운영자가 더 빨리 움직이도록 도와야 하지만, 위험한 단계를 가려서는 안 됩니다.
그래서 이 로드맵은 아래 원칙을 유지합니다.

1. 명확성이 자동화보다 먼저입니다.
2. 검토와 발행은 분리합니다.
3. 실제 공개 결과 검증을 생략하지 않습니다.
